  <name>Task 1: Add click handlers for bidirectional drill-down</name>
  <files>src/components/visualizations/D3NetworkGraph.tsx</files>
  <action>
1. Add onClick handler to node circles:
   - When node is clicked and NOT currently selected: call selectNode(nodeId, edges)
   - When same node is clicked again: call clearSelection() (toggle behavior)
   - This works identically for pathogens and antibiotics (bidirectional)

2. Add background click handler to clear selection:
   - Add onClick to the SVG element itself
   - When SVG background is clicked (not a node): call clearSelection()
   - Use stopPropagation on node clicks to prevent SVG handler from firing

3. Add visual indicator for selected node:
   - Selected node gets a highlight ring (additional circle with stroke, no fill)
   - Ring radius = node radius + 4
   - Ring stroke color = white or gold (#FCD34D) for visibility

AVOID: Complex state machines - keep it simple: selected or not selected. The useNetworkSelection hook from 05-01 handles the logic.
  </action>

5. Click different node while one selected - selection switches to new node
  </verify>

<task type="auto">
  <name>Task 2: Add smooth CSS/D3 transitions for fade effect</name>
  <files>src/components/visualizations/D3NetworkGraph.tsx, src/styles/networkAnimations.css</files>
  <action>
1. Add CSS transition to network elements:
   - Create src/styles/networkAnimations.css with:
     ```css
     .network-node { transition: opacity 300ms ease-in-out; }
     .network-edge { transition: opacity 300ms ease-in-out, stroke-opacity 300ms ease-in-out; }
     .network-highlight-ring { transition: opacity 200ms ease-out; }
     ```
   - Import this CSS file in D3NetworkGraph.tsx

2. Add className to SVG elements:
   - circle elements: className="network-node"
   - line elements: className="network-edge"
   - highlight ring: className="network-highlight-ring"

3. Optional enhancement: Connected nodes get subtle scale or glow effect:
   - Add transform: scale(1.1) or filter: drop-shadow for connected nodes
   - This provides extra visual emphasis beyond just opacity
   - Keep it subtle - don't distract from the relationships

AVOID: D3.transition() for the opacity changes - CSS transitions are simpler and work better with React's declarative rendering. D3 transitions would fight with React's re-renders.
  </action>

3. Switch selection - old connections fade while new ones brighten smoothly
4. No jarring or instant opacity changes
