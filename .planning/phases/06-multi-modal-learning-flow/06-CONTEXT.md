# Phase 6: Multi-Modal Learning Flow - Context

**Gathered:** 2026-01-07
**Status:** Ready for planning

<vision>
## How This Should Work

The app opens to a central hub with three big, visual cards — one for each learning mode: Visual Network, Comparison, and Quiz. Each card is a clear, inviting entry point that makes the mode's purpose obvious at a glance.

When exploring antibiotics in any mode, the context carries with you. If you're looking at Vancomycin in the network visualization and want to compare it to something else, switching to Comparison mode keeps Vancomycin selected. The modes feel connected, not siloed.

Navigation between modes should feel seamless and instant. A "hub & spoke" model where you can always get back to the central dashboard and jump to whichever mode fits your current learning goal.

</vision>

<essential>
## What Must Be Nailed

- **Clear purpose for each mode** - When landing on the hub, immediately understand what each mode offers and when to use it
- **Context preservation** - Selected antibiotics/pathogens carry across mode switches (no "starting over")
- **Seamless switching** - Getting between modes feels instant and natural, never jarring

</essential>

<boundaries>
## What's Out of Scope

- Progress tracking / "how am I doing" displays — that's Phase 8
- Advanced recommendations ("you should review X") — that's Phase 8
- Spaced repetition / quiz scheduling — explicitly deferred from roadmap
- Analytics dashboards — Phase 8 scope

This phase is purely about connecting the existing modes, not adding intelligence or tracking to them.

</boundaries>

<specifics>
## Specific Ideas

No specific requirements — open to standard approaches for navigation patterns, mode indicators, and return-to-hub mechanics.

</specifics>

<notes>
## Additional Context

This phase brings together all the work from Phases 2-4 (Visual Network, Comparison Interface, Quiz System). The three modes are already built and functional — this phase focuses entirely on the connective tissue that makes them feel like one cohesive learning experience rather than three separate tools.

</notes>
