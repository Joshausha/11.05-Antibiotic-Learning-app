<task type="auto">
  <name>Task 1: Create LearningHub component with 3 mode cards</name>
  <files>src/components/LearningHub.tsx</files>
  <action>
Create a new LearningHub.tsx component with:

1. Three large, visually distinct cards for each learning mode:
   - **Explore** (Network visualization) - icon: Network, color: blue, navigates to 'visualizations'
   - **Compare** (Side-by-side comparison) - icon: GitCompare, color: green, navigates to 'comparison'
   - **Quiz** (Test your knowledge) - icon: Brain/BookOpen, color: purple, navigates to 'quiz'

2. Each card includes:
   - Large icon at top
   - Mode name as heading
   - 1-2 sentence description explaining when to use this mode
   - "Start" or "Explore" button

3. Card descriptions (per 06-CONTEXT.md clarity requirement):
   - Explore: "See how antibiotics connect to pathogens. Discover coverage patterns through interactive network visualization."
   - Compare: "Put antibiotics side-by-side to understand similarities and differences in coverage, formulation, and mechanism."
   - Quiz: "Test your antibiotic knowledge with board-prep style questions. Get detailed explanations for every answer."

4. Responsive layout: Grid that shows 3 cards in row on desktop, stacked on mobile.

Use TypeScript, import setActiveTab type from app.types.ts. Accept onNavigate: (tab: TabType) => void as prop.

Do NOT use external animation libraries. Simple Tailwind hover states are sufficient.
  </action>
  <verify>TypeScript compiles without errors for the new component: `npx tsc --noEmit src/components/LearningHub.tsx` or check via build</verify>
  <done>LearningHub.tsx exists with 3 mode cards, each with icon, title, description, and navigation button</done>
</task>

  <action>
Update the app to use LearningHub as the default landing:

1. In app.types.ts:
   - Add 'hub' to TabType union (before 'learn')

2. In App.tsx:
   - Add lazy import for LearningHub
   - Change default activeTab from 'learn' to 'hub' in destructuring
   - Add new tab case for 'hub' that renders LearningHub with onNavigate={setActiveTab}
   - Keep existing 'learn' tab for backward compatibility (it can remain as an alternate entry)

3. LearningHub navigation:
   - Pass setActiveTab as the onNavigate prop
   - When user clicks a mode card, navigate to that tab

This establishes the hub & spoke model from 06-CONTEXT.md - hub is the center, modes are spokes.
