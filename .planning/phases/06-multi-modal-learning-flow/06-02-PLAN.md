  <name>Task 1: Create SharedSelectionContext for cross-mode selections</name>
  <files>src/contexts/SharedSelectionContext.tsx, src/types/app.types.ts</files>
  <action>
Create a new React Context for shared selections that persist across mode switches:

1. In app.types.ts, add:
```typescript
export interface SharedSelection {
  selectedAntibiotics: string[];  // antibiotic names/IDs
  selectedPathogens: string[];    // pathogen names/IDs
  lastSelectedAntibiotic: string | null;  // most recent selection for single-select modes
  lastSelectedPathogen: string | null;
}
```

2. Create src/contexts/SharedSelectionContext.tsx:
   - SharedSelectionProvider wraps app (inside AppProvider)
   - State: SharedSelection object (per single-object pattern from 02-02)
   - Actions: addAntibiotic, removeAntibiotic, clearAntibiotics, setAntibiotics
   - Same for pathogens: addPathogen, removePathogen, clearPathogens, setPathogens
   - useSharedSelection hook for consuming context

3. Initial state: empty arrays, null last selections

This context is SEPARATE from AppContext to maintain separation of concerns. AppContext handles navigation/UI state, SharedSelectionContext handles cross-mode learning state.

Do NOT modify existing components yet - just create the context infrastructure.
  </action>
  <verify>TypeScript compiles: `npx tsc --noEmit src/contexts/SharedSelectionContext.tsx`</verify>
  <done>SharedSelectionContext.tsx exists with provider, hooks, and SharedSelection interface</done>
</task>

Integrate the shared selection context and add hub navigation:

1. In App.tsx:
   - Import SharedSelectionProvider
   - Wrap AppContent with SharedSelectionProvider (inside AppProvider)

2. In Header.tsx:
   - Add "Hub" or home icon as first navigation item that navigates to 'hub' tab
   - This provides the "always get back to central dashboard" from 06-CONTEXT.md

3. In ComparisonModeContainer.tsx:
   - Import useSharedSelection hook
   - On mount: if sharedSelection.selectedAntibiotics has items, pre-select them in comparison state
   - When user selects antibiotics in comparison, update sharedSelection via addAntibiotic/setAntibiotics
   - This creates bidirectional sync between comparison mode and shared state

The key insight: ComparisonModeContainer already has its own selection state. We're adding a sync layer that:
- Reads from shared context on mount (to inherit selections from other modes)
- Writes to shared context on change (to propagate to other modes)

Do NOT modify D3NetworkGraph yet - visual selection sync can be Phase 7 polish.
